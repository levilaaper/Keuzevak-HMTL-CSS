Stap 2

Maak twee bestanden aan; het eerste bestand is het html bestand waarin je HTML code gaat schrijven, dit bestand bevat dus de inhoud van je website. Het tweede bestand is het css bestand waarin je de CSS code gaat schrijven, dit bestand bevat dus hoe je website eruit moet komen te zien.

HTML staat voor 'HyperText Markup Language', er bestaat per pagina 1 html bestand, de homepagina noem je altijd "index.html". Elke computer en server weet dat door het woord "Index" dat dit het eerste bestand is waar naar gekeken moet worden.

CSS staat voor 'Cascading Style Sheets', een CSS bestand moet je zelf altijd koppelen zodat Chrome weet in welke opmaak hij de index.html moet weergeven, je noemt dit bestand "style.css"

Stap 3

Github Desktop zorgt ervoor dat je verschillende versies van je code online worden bewaard. Hierdoor hoef je nooit bang te zijn dat je persongelijk al je code permanent beschadigd door typo's. Ook kun je door Github andere mensen je code laten zien als je er voor kiest om je code online weer te geven. Volg de instructies van onderstaand uitleg om Github Desktop op te zetten.

Github Desktop setup tutorial door Broezer

Stap 4

Nu je twee lege bestanden in je Code open hebt staan en je geschreven code wordt opgeslagen in de cloud kunnen we beginnen met schrijven. Hier zie je simpel voorbeeld van hoe je de twee bestanden opzet en vult met eerste code. Aan de linker kant staat de "index.html" waarin je declareert door '!doctype html' dat het een HTML bestand is. Vervolgens link je het "style.css" bestand door de zin 'link rel="stylesheet" type="text/css" href="style.css" in de head van je HTMl toe te voegen.'

Stap 5

Als je iets aanpast in Code maar het resultaat niet meteen ziet in Chrome, gaat er dus ergens iets mis, om uit te zoeken wat er mis gaat kun je met de muis op een element gaan staan, rechter muisknip indrukken en "element inspecteren" selecteren. Er zal een scherm verschijnen en je kunt hierin eenvoudig de code nalezen waar het mis is gegaan.

"Element Inspecteren" werkt op alle websites! Als je dus bent benieuwd hoe andere websites er "onder de motorkap" uitzien kun je deze functie gebruiken om rond te kijken.

Stap 6
